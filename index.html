反向传播中链式求导与误差的理解

Posted on 2019-09-19

问题描述

人工的神经网络可以看成是多个logistic回归的叠加，在用梯度下降求解logistic回归代价函数时，我们直接用偏导的公式得到梯度:

那么对于神经网络来说，即使代价函数更为复杂，我们也可以通过链式求导法则去求得每层参数的梯度，从而完成梯度下降的更新，为什么要引入反向传播算法与误差的概念？

理解

首先可以肯定的是反向传播算法的本质依然是使用链式求导法则。但没有直接使用链式求导，而用反向传播算法和误差的原因我认为有两点:

1. 模型的通用化

对于一个较深的神经网络，求导每一层参数的梯度的公式都不一样，比如第一层的梯度一定是一个非常长串的公式。反向传播中误差δ只是一个记号，暂存了求导时的中间变量。虽然不清楚反向传播每一层的具体细节做了什么，但可以理解为每一层都用相同的模式处理，就是用该层的误差去更新该层的权重。

2. 缩减计算成本

回顾高数中的链式求导，可以知道求导过程中存在大量的重复计算，求解链末端的导数需要将前面的导数都计算一遍。引入δ误差存储中间结果，极大的避免了重复计算。

假设已经安装好git，此时我们如何使用git创建版本库来管理我们的项目？打开git bash并且输入以下代码:

git init 初始化当前文件夹为一个版本库，于是git可以控制在此文件夹或其子文件夹的内容。（文件夹的内容不需要为空）

现在一个项目已被git管理，我们现在添加一个文件learngit.txt并修改它。

git status 查看当前工作区的状态，git add 将当前工作区修改的文件添加到缓冲区(git add . 添加当前文件夹下所有文件)，git commit 将缓存区的修改全部提交到master分支中，-m 是这次提交的描述（description），git diff 用于查看版本库和未提交到缓存区的文件的不同（即在 git add操作之前）。

经过多次commit提交后，git上会保存多个版本，如果我们想回到上一个版本:

git log ：查看版本提交记录， head^为回到上一个版本 ^^head回到倒数第二个版本，注意回到前面的某个版本Vo后，Vo后面的log信息将不存在。git reset —hard <……> …是用git log 查出来的版本号，输入前几位git即可搜索，此命令可以回到该版本号的版本。

现在又想回到最新的版本。

get reflog 可以查看版本操作历史，可以列出所有的版本号。

如何撤销修改

  • 撤销未进入换成区的修改

  • 已经提交到版本库，commit后自然只有回退版本了。

删除文件
用 rm \ 删除的文件可以用 git checkout — \ 恢复，git rm \删除的文件只能通过回退版本。
